# 랜딩페이지 프로젝트

HTML과 CSS를 활용하여 웹표준과 시멘틱 마크업을 고려한 랜딩페이지를 제작한 프로젝트입니다.  
피그마 시안을 기반으로 레이아웃을 구현하고, 반응형 및 접근성을 고려하여 구성했습니다.

---

## 🔗 배포 링크
- https://sueaion.github.io/mini-p2/

---

## 📁 폴더 구조
mini-p2/
├─ index.html
├─ styles/
│ └─ style.css
├─ images/
│ └─ (이미지 파일들)
└─ README.md
## 🛠 사용 기술
- HTML5
- CSS3 (Flexbox)
- GitHub Pages

---

## ✨ 주요 구현 사항
- 시멘틱 태그(`header`, `main`, `section`, `article`, `footer`)를 활용한 구조 설계
- `ul > li` 구조를 사용한 네비게이션 마크업
- `aria-label`, `alt`, `label` 등을 활용한 접근성 고려
- HERO / FEATURE / GALLERY / NEWSLETTER 섹션 구성
- Flexbox를 활용한 레이아웃 정렬
- Top 버튼을 활용한 사용자 편의 기능
- 768px 기준 반응형 레이아웃 구성

---

## 📱 반응형 기준 (768px)
- 768px은 태블릿 환경(세로 기준)을 고려한 브레이크포인트로 설정하였습니다.
- 해당 해상도 이하에서는 레이아웃 간격과 정렬을 조정하여 가독성을 유지했습니다.

---

## 🎨 스타일링 특징
- 반복적으로 사용되는 색상은 CSS 변수로 관리하여 유지보수를 고려했습니다.
- 전역 태그 선택자 사용을 지양하고, 클래스 기반 스타일링을 적용했습니다.
- `flex`를 중심으로 레이아웃을 구성하여 `float` 사용을 최소화했습니다.
- `width: 100vw` 사용을 지양하고 `max-width`와 `margin: 0 auto`로 레이아웃을 제어했습니다.

---

## ♿ 접근성 & 웹표준 고려
- 모든 이미지에 의미 있는 `alt` 속성 작성
- 입력 요소에 `label`과 `sr-only` 텍스트 제공
- 네비게이션 영역에 `aria-label` 추가
- HTML 문서 내 태그 닫힘 및 속성 오류 수정

---

## 🧠 작업 방식
- 초기 구현 후 웹표준·접근성 관련 피드백을 반영하여 마크업 구조를 개선했습니다.
- 기능 단위 커밋보다는, 수정 사항을 정리한 뒤 최종본 기준으로 커밋했습니다.

- CSS 중복 코드 정리 및 유지보수성 개선
